## Description

Build a real-time fraud detection engine that uses ML models trained on Meridian's transaction data to identify fraudulent payments before settlement. Replace the current rules-based system (217 static rules, last updated Oct 2025) with a gradient-boosted model that scores every transaction in under 50ms.

Our current fraud loss rate is 0.12% ($2.52M annually on $2.1B TPV). Industry average for our segment is 0.08%. We're leaving $840K on the table every year — and more importantly, every fraud incident erodes merchant trust.

The model will run as a sidecar service to the payment processing pipeline, scoring transactions in real-time and flagging high-risk payments for manual review or automatic decline based on merchant-configurable thresholds.

## Technical Approach

- **Model**: XGBoost ensemble trained on 18.4M historical transactions (Jan 2025 – Jan 2026)
- **Features**: 147 features including velocity checks, device fingerprinting, geographic anomalies, merchant category patterns
- **Serving**: ONNX runtime deployed as gRPC sidecar, Kubernetes pod affinity with payment-service
- **Feedback loop**: Manual review outcomes feed back into training pipeline every 72 hours
- **Shadow mode**: Running in parallel with rules engine since Feb 24; cutover planned for Mar 14
