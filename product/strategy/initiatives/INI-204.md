## Description

Add multi-currency payment processing to Meridian Pay, starting with EUR, GBP, and CAD. This is the single most requested feature from enterprise prospects and the primary reason for our largest churn event in Q3 2025 (DataWeave, $287K ARR, switched to Adyen).

Multi-currency requires deep changes: new settlement flows, FX rate management, currency-aware ledger entries, updated merchant invoicing, and regulatory compliance for cross-border transactions. We're partnering with CurrencyCloud for FX execution and using their real-time rate feed.

Phase 1 (Q1): Architecture, FX integration, EUR support in sandbox
Phase 2 (Q2): GBP + CAD, production launch, merchant migration tools

## Technical Approach

- **FX Provider**: CurrencyCloud — real-time rate feed, automated FX execution, 34 currencies available (we're starting with 3)
- **Ledger Changes**: Every transaction now carries `processing_currency`, `settlement_currency`, and `fx_rate` fields
- **Settlement**: New settlement engine handles multi-currency batching, FX execution at T+1, and merchant payout in preferred currency
- **Database**: New `currency_rates` table with 15-minute snapshots; `transactions` table extended with currency fields
- **API Changes**: `/v1/payments` accepts optional `currency` parameter; backward-compatible (defaults to USD)
