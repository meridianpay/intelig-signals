## Description

Achieve PCI DSS 4.0 compliance before the March 31, 2026 deadline. PCI DSS 4.0 introduces 64 new requirements compared to v3.2.1, including mandatory multi-factor authentication for all access to cardholder data, enhanced encryption requirements, and new tokenization standards.

We passed our PCI DSS 3.2.1 audit in September 2025, but 4.0 has significant new requirements. Our QSA (Coalfire) identified 6 gaps in their preliminary assessment in January 2026. We've closed 4, with 2 remaining — both related to the new tokenization requirements (Requirement 3.5.1.2).

Non-compliance is not an option. It would mean loss of card network partnerships and the ability to process payments. This is existential.

## Technical Approach

- **Tokenization**: Migrating from software-based AES-256 tokenization to HSM-backed (Thales Luna Network HSM 7) tokenization
- **Key Management**: Automated key rotation every 90 days via HSM API, split knowledge procedures for master keys
- **MFA**: Okta Verify for all CDE access, hardware tokens (YubiKey) for production database access
- **Logging**: All CDE access logged to immutable audit trail (CloudWatch Logs → S3 with Object Lock)
- **Penetration Testing**: Completed Feb 18 by NCC Group — 0 critical/high findings, 3 medium (all remediated)
