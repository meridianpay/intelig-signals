## Description

The current merchant portal was built in 2022 using Angular 12 and has accumulated significant tech debt. Merchant support tickets related to portal usability average 4.7 per merchant per month. The transaction search takes 12 seconds to load. The settlement reports page has a 34% bounce rate. It's embarrassing.

We're rebuilding on React 18 + TypeScript with a component library based on Radix UI. The new portal will be faster (sub-2s page loads), more intuitive (reducing support tickets by 50%), and extensible enough to support multi-currency dashboards, fraud analytics, and API key management.

## Technical Approach

- **Frontend**: React 18, TypeScript 5.3, Vite, TanStack Query, Radix UI + custom design system
- **State Management**: TanStack Query for server state, Zustand for client state
- **Backend**: New BFF (Backend for Frontend) service in the monolith, purpose-built for portal queries
- **Search**: Elasticsearch 8.x for transaction search (replaces PostgreSQL full-text search)
- **Auth**: Existing merchant auth (JWT) with new RBAC layer for team management
